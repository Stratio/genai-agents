### 0.1.0 (2026-05-05)

First release of the `genai-agents` monorepo: seven agents, the shared-skills system, multi-platform packaging and internationalization.

* **Agents**: `data-analytics` (full BI/BA with multi-format report generation), `data-analytics-light` (chat-oriented variant for Claude AI Projects and Claude Cowork), `semantic-layer` (building and maintaining semantic layers in Stratio Governance), `data-quality` (coverage assessment and quality-rule generation), `governance-officer` (combined semantic-layer + data-quality), `skill-creator` (interactive skill design) and `agent-creator` (interactive agent design).
* **Data and governance shared skills**: `propose-knowledge`, `explore-data`, `stratio-data`, `stratio-semantic-layer`, `create-technical-terms`, `create-ontology`, `create-business-views`, `create-sql-mappings`, `create-semantic-terms`, `manage-business-terms`, `create-data-collection`, `build-semantic-layer`, `assess-quality`, `create-quality-rules`, `create-quality-schedule` and `quality-report`.
* **Visual-craftsmanship family**: deliverable skills (`pdf-writer`, `docx-writer`, `pptx-writer`, `xlsx-writer`, `web-craft`, `canvas-craft`) and their companion readers (`pdf-reader`, `docx-reader`, `pptx-reader`, `xlsx-reader`), all following the guidance-first pattern and delegating visual identity tokens to the centralized `brand-kit` skill (ten curated themes, extensible by clients).
* **Stratio integration**: `stratio_data` and `stratio_gov` MCPs, long-running-task polling protocol, OpenSearch fallback for searches, and shared guides (`stratio-data-tools.md`, `stratio-semantic-layer-tools.md`, `quality-exploration.md`, `visual-craftsmanship.md`).
* **Multi-platform packaging**: generic root scripts (`pack_claude_code.sh`, `pack_opencode.sh`, `pack_shared_skills.sh`) and agent-specific scripts (`pack_claude_ai_project.sh`, `pack_claude_cowork.sh`), orchestrated by `bin/package.sh` and `make package`.
* **Internationalization**: English as the primary language with a full `es/` overlay (AGENTS.md, SKILL.md, guides, READMEs, templates); `bin/check-translations.sh` verifies parity and all pack scripts accept `--lang <code>`.
* **Runtime and CI/CD**: sandbox-provided venv (`/opt/venv` on Stratio Cowork, local `.venv` in dev), per-agent `requirements.txt` with the analytical stack (pandas, scipy, scikit-learn, matplotlib, seaborn, plotly) and deliverable-generation stack (reportlab, pypdf, python-docx, python-pptx, openpyxl, markdown). Jenkins pipeline (`Jenkinsfile`, `Makefile` and `bin/` utilities).
* **Shared-skill READMEs**: all 28 shared skills now ship a user-facing `README.md` (and `es/` counterpart). The 17 skills that had no README got one; the 11 existing READMEs were harmonized with companion-skill cross-references, MCPs section, corrected font-bundle details and a brand-kit tone-family table. Template covers: what it does, when to use, dependencies (skills / guides / MCPs / Python / system), bundled assets and notes.
* **`manage-business-terms` domain-selection rule**: `create_business_term` now defaults to the semantic domain (`semantic_<x>`) when one exists, correcting a systematic misclassification where terms were filed under "Technical Knowledge" instead of "Semantic Knowledge". The skill documents the three legitimate technical-domain edge cases, adds a prefix-consistency rule for `related_assets`, and includes worked examples. `stratio-semantic-layer-tools.md` gains a dedicated "Business terms domain selection (post-publication)" rule with an explicit scope guard preventing the heuristic from leaking into the construction pipeline. Triage rows in `semantic-layer` and `governance-officer` agents expanded from one example to eight (KPI, glossary entry, metric, domain concept, acronym, dictionary definition, asset-linked concept).
* **`cowork-api` shared-skill and direct upload from `skill-creator` / `agent-creator`**: new `cowork-api` shared-skill (router with per-capability sub-files under `tasks/`) covers registering skill bundles (`POST /v1/agents/skills/bundle/import`) and agent bundles (`POST /v1/agents/bundle/import`) directly from inside the Stratio sandbox; new `external-api-calls` shared-skill-guide centralizes mTLS env vars, pre-check, and curl/Python templates for any skill that calls a Stratio REST API. `skill-creator` and `agent-creator` delegate their end-of-packaging upload offer to the new skill, eliminating the manual download/upload roundtrip through the web UI.
* **Glossary-driven instruction enrichment for semantic-layer skills**: new `get_glossary_instructions` MCP tool (`stratio_gov`) exposes the GenAI instruction-typed business terms from the data dictionary as a consultable, filterable resource (per-phase and global). The `build-semantic-layer` orchestrator pre-loads enriched instructions once at the start of the pipeline and the four phase skills (`create-technical-terms`, `create-ontology`, `create-sql-mappings`, `create-semantic-terms`) ask the user — before invoking any creation tool — whether to apply specific glossary instructions, specific + globals, an external file, or skip enrichment. The chosen text is consolidated and passed as `user_instructions`. The shared guide `stratio-semantic-layer-tools.md` gains a dedicated §11 documenting the workflow; `semantic-layer` and `governance-officer` agents reference it from their capabilities and user-interaction sections.
* **`semantic-layer` and `governance-officer`: SQL mapping validation and semantic layer sanity-check**: agents can now validate SQL mappings with sample data before publishing and run end-to-end sanity checks on the published `semantic_<domain>` — without leaving the agent or switching to data-analytics. `build-semantic-layer` gains an optional pre-publication validation step (uses `sql_mapping` from `create_sql_mappings.processed_views` directly, no extra round-trip) and a post-publication §7 with propagation-latency handling (60 s after `publish_business_views`, 90 s after `create_semantic_terms`). `create-sql-mappings` gains §6.5 with the same per-view validation flow. `stratio-data-tools.md` gains §3.5 "Showing query results to the user": default cap of 10 rows in chat (50 when the user explicitly asks for more), edge-case rules for 0 rows and exact-cap counts. `semantic-layer/opencode.json` enables `execute_sql` and `generate_sql` (previously denied); `governance-officer` removes the erroneous restriction that data tools were unavailable in the semantic-layer workflow.
* **`agent-creator` / `skill-creator`: packaging cleanup, mandatory Cowork deployment, and host-skill isolation**: five behavioural fixes applied across both agents, `cowork-api`, and the shared packaging/API guides (EN + ES). (1) Packaging now uses a workdir-local `.pack-staging/` dir instead of `mktemp -d` (avoids `/tmp/` inaccessibility in the Stratio sandbox), with centralised cleanup and a pre-delivery check that lists artefacts that must / must-not be present. (2) Bundle upload to Stratio Cowork is now a mandatory workflow step, not an opt-in offer: all sandbox-conditionality language ("only offered when running inside the Stratio sandbox", "skip silently if pre-check fails") is removed; a routing rule in Phase 0 with `MUST load /cowork-api` fires for any upload/import/deploy/publish/register intent at any phase. (3) `cowork-api` description front-loaded with user-facing verbs (`upload`, `import`, `deploy`, `publish`, `register`, `send`, `push` + Spanish equivalents) for reliable auto-activation; pre-check reframed as environment health-check throughout SKILL.md, tasks, and `external-api-calls.md`. (4) New `CRITICAL` block in `agent-creator` sección 1 and "Skill decomposition rules" section in `agent-designer` prevent host-runtime skills (e.g. `caveman`, OpenCode plugins) from being proposed as candidates for the agent being built; Phase 1 Q6 default is now "no external skills". (5) Claude Code references removed from `skill-creator` AGENTS.md and all four README files; only `pack_opencode.sh` / `pack_stratio_cowork.sh` are listed as packaging scripts for these two agents.
