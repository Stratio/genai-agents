# Changelog

## 0.1.0 (upcoming)

* [ROCK-NA] pack_opencode.sh / pack_claude_code.sh: fix `shared-skills/<name>/` path corruption in the rewrite pass. The previous regex did not exclude `-` from the preceding-char class, so `shared-skills/docx-writer/scripts` was rewritten to `shared-.opencode/skills/docx-writer/scripts` (and the claude equivalent), which did not resolve at runtime and forced agents to improvise `Path(__file__).resolve().parents[N]` tricks. The fix adds `-` to the negated class and adds a dedicated first pass that rewrites `shared-skills/<name>/` → `.opencode/skills/<name>/` (or `.claude/skills/<name>/`) so packaged SKILL.md ships with runnable `sys.path.insert(0, ".opencode/skills/docx-writer/scripts")` / `sys.path.insert(0, ".claude/skills/docx-writer/scripts")`. Smoke-tested with the packaged runtime topology for all four agents (data-analytics, data-quality, governance-officer, semantic-layer).
* [ROCK-NA] docx-writer enhancements: `DOCXBuilder` gains `add_html_block(html)` with inline base64 image decoding and rich HTML tag coverage (`<p>`, headings, lists, tables, `<figure>`, `<div class="callout">`, `<blockquote>`), and `add_markdown_block` now delegates to it so markdown input supports inline images too. `add_table(cell_colors=...)` paints individual cells (e.g., status/priority columns) with semantic colours. `add_heading` always sets `keep_with_next=True` and accepts an optional `page_break_before`. Nested inline formatting (`**bold with *italic* inside**`) renders bold-italic runs correctly. Four editorial tone presets (`default`, `technical-minimal`, `executive-editorial`, `forensic`) added to `palette.py` — usable directly from any `DOCXBuilder(aesthetic_direction={"tone": ...})` call. `shared-skills/quality-report/scripts/quality_report_generator.py` adds inline status/priority cell colouring (OK/KO/WARNING and CRITICO/ALTO/MEDIO/BAJO) to the DOCX output using python-docx directly — parity with the HTML/PDF path that previously had them. 9 new tests in docx-writer (html_block, cell_colors, tones, nested bold, missing figure, palette RGB override). Skill stays self-contained per the monorepo rules; no cross-skill imports in runtime.
